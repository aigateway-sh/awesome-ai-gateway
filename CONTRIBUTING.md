# Contributing to Awesome AIgateway

Thanks for helping curate this list. We want the bar high enough that developers can trust anything they find here.

## What we accept

- **Tools, libraries, and frameworks** that work with AIgateway out of the box (OpenAI-compatible providers, MCP clients, or direct SDK users).
- **Starter templates and examples** that a developer can run in under 5 minutes.
- **Production case studies** with enough detail to be instructive (architecture, numbers, lessons).
- **Tutorials, blog posts, and videos** that teach something non-obvious.

## What we decline

- Promotional posts with no substance.
- Broken, abandoned, or alpha-only projects.
- Duplicates of already-listed tools.
- Projects without public documentation.
- Paid-only products without any free tier or trial.

## Format

Every entry must follow this shape:

```markdown
- [**Project Name**](https://url) — one-line description. Why a dev would care.
```

- **Project Name** goes in bold inside the link.
- The em-dash (`—`) separates name from description.
- Keep the description to one sentence. Multi-sentence descriptions get rejected.
- End with a period.

Example:

```markdown
- [**Mastra**](https://mastra.ai) — TypeScript agent framework; configure the OpenAI provider with our base URL.
```

## Pull request checklist

- [ ] Added under the most specific category.
- [ ] Description is one line and ends with a period.
- [ ] Link resolves (no 404s, no paywalls blocking a landing page).
- [ ] Repo has a README or equivalent public docs.
- [ ] If it's a library, it's been updated in the last 6 months (or is explicitly "stable / maintenance mode").
- [ ] Categories are still alphabetized within each section (sub-section ordering is intentional where present — preserve it).

## Reviewing

Maintainers aim to review PRs within a week. A `needs-one-line` or `needs-more-info` label means we like the idea but need a clean-up before merge.

## Code of conduct

Be kind. Attack the idea, not the person. Follow the [Contributor Covenant](https://www.contributor-covenant.org/version/2/1/code_of_conduct/).
